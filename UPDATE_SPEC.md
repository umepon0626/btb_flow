# **バスケットボール・シュートフロー解析アプリ：改善・機能拡張設計書**

本ドキュメントは、Google MediaPipe PoseLandmarkerを活用したバスケットボールのシュート動作解析アプリにおける、現状の課題（トラッキングの不安定さ、骨格推定の精度）に対する解決策、および詳細な動作分析のための新規メトリクス実装設計をまとめたものです。

## **1\. 精度向上とトラッキングの安定化策**

### **1.1 トラッキングのロジック改善**

バスケットボールのように選手が激しく交差する環境では、重心距離のみの判定ではIDスイッチが発生しやすくなります。以下の改善を推奨します。

* **移動予測（Velocity）の導入:** 前フレームの移動ベクトルから現フレームの予想位置を計算し、その位置に最も近い人物を追跡対象とします。  
* **領域重なり（IoU）の活用:** 人物のバウンディングボックス（BBox）の面積の重なり度合いを評価に加え、交差時の誤認を防ぎます。

### **1.2 骨格認識精度の向上**

* **モデルのアップグレード:** pose\_landmarker\_lite.task から pose\_landmarker\_full.task または heavy への変更。これにより、四肢の重なりに対する推論精度が劇的に向上します。  
* **トップダウン・クロップ解析:** トラッキング対象が確定した後、その人物の周囲のみをクロップ（切り出し）して拡大し、再度Poseモデルに入力することで実質的な解像度を向上させます。

## **2\. データ平滑化（スムージング）の実装**

MediaPipeの出力値に含まれるジッター（小刻みな震え）を抑制するため、**指数移動平均（EMA: Exponential Moving Average）**を実装します。

// スムージング計算式  
// SmoothedValue \= (PrevSmoothedValue \* (1 \- alpha)) \+ (CurrentValue \* alpha)  
// alpha: 0.4 ～ 0.6 を推奨

## **3\. 解析メトリクスの詳細設計**

シュートフォームを定量評価するために以下の6つの指標を新たに計測・記録します。

| メトリクス名 | 使用ランドマーク（Index） | 算出の目的   |
| :---- | :---- | :---- |
| 右手首の相対位置 | 腰中央, 右手首(16) | ボールを上げる「リフト」の軌道可視化 |
| 腰の前屈角 | 肩(12), 腰(24), 膝(26) | タメ（ディップ）の深さと上体の傾き計測 |
| 右肘の角度 | 肩(12), 肘(14), 手首(16) | セットポイントおよびフォロースルーの伸び評価 |
| 右腕のヨー角 | 左肩(11), 右肩(12), 手首(16) | 体の正面に対する脇の開き・肩の入り具合 |
| 膝の前屈角 | 腰(24), 膝(26), 足首(28) | 下半身のパワー生成プロセスの分析 |
| リリース角度 | 右肩(12), 右手首(16) | リリースの高さと放物線のアーチの推定 |

## **4\. 技術的実装のポイント**

### **4.1 体の向き（胸の向き）の考慮**

MediaPipeのZ座標を活用し、X-Z平面上で肩のラインとカメラのなす角を算出することで、被写体がリングに対してどれだけ「ターン」しているかを判定します。

### **4.2 3Dベクトルによる角度計算**

単純な2D座標の角度計算ではなく、x, y, zの3次元空間におけるベクトルの内積を用いて角度を算出することで、カメラの角度に左右されにくい安定したデータ取得を目指します。
